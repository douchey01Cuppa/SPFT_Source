Configure File

 

·  Configure File Structure

In console mode configure file, general setting and command setting are included.

All the values in the configure file comes from the configuration in Flash Tool UI, and the values could be customized with correct format if necessary.

 

·   General Setting

·         connect-args: specific the connection relative configuration.

–     timeout_ms: configure connection timeout limitations

–     com_port_name: configure which com port will be used when scan

–     conn_stage: configure which stage will be connected to, brom, 1st DA and 2nd DA are provided.

–     baud_rate: configure baudrate when connection.

–     com_port_type: which type of com port will be connected, USB-PCIE and UART are provided.

–     flash-xml: speficic the flash xml file path..

–     auth-file: specific the authentication file path .

–     cert-file: specific the certification file path .

·         runtime-parameter: specific the runtime parameter setting.

–     checksum_level: specific the checksum level, if value is 1, none checksum will be performed, 2 means the USB checksum will be performed, 4 means Storage checksum will be performed, and 6 means both USB checksum and Storage checksum..

–     battery_status: configure target power supply, if value is 2, power supply comes from USB, and battery is optional; if value is 1, power supply comes from battery and battery is required; if value is 0, power supply from battery or USB VBUS is auto detected by DA.

–     da_log_level: DA log level is used to configure tracing log level. DEBUG, INFO, WARN and ERROR are available.

–     da_log_channel: DA log channel is used to configure tracing log channel. USB and UART are available.

–     system_os: specific the current system platform, WINDOWS and LINUX are available.

·         log-info: console mode log switch and time setting to delete log file regularly, it is generated by ui.

log_on: console mode log switch, enable log if true, otherwise false.

log_path: the directory in which the log files has been stored.

clean_hours: the time setting to delete log files regularly, the unit is hours.

 

 

·  Command Setting

Command Setting includes basic configurations for the operation.

·     Format

Using ERASE-FLASH cmd to do format operation, include auto format and manual format.

offset: The start address offset from the beginning of selected partition.

length: The data length wanted to read.

partition: specific the region of format data.

 

 

 

 

·     Download

FlashTool can export three different download command setting according to specific download scene seleted.

The three download scene are:

–     Download Only

Using WRITE-PARTITIONS command to do download only opertition.

source-file: specify the scatter xml file .

flash_list: list the partitions and its file path which need download to device.

 

 

–    Format All + Download

Using ERASE-FLASH to do whole format and FLASH-ALL 2.0 to do download operation.

 

 

–     Firmware Upgrade

Using FLASH-UPDATE 2.0 command to do fireware upgrade operation.

 

 

 

·     Readback

Using READ-FLASH to do Manual Readback. .

 

 

And READ-PARTITION to do Auto Readback .

 

 

·  Configure File Generation

Please make sure all the configurations mentioned in “General Configuration” has been ready before generate the configure file.

 

·  Select Operation

Configure File for Console Mode could be generated by Flash Tool GUI.

Press “[Menu] File-> [Submenu] Export Console Mode XML” to select the operation expected.

Support Operations: Format, Download, Readback.

 

 

·  Save File

Save the configure file with specific file path and name.

Default path is Flash Tool path and default file name is [operation].xml.
